# PuttNavi 距離測定精度テスト計画書

## 目的
距離推定の精度が悪い原因が「カメラ映像からの初速計算のブレ」なのか、「Stimp値などの物理モデルのズレ」なのかを切り分け、適切な補正アルゴリズムを導入するためのデータを取得します。

## テストの準備
1. **メジャー付きのパターマット**（または床にメジャーを敷いた環境）を用意します。
2. アプリ（PuttNavi）を起動し、カメラを適切な位置（床直置き、または三脚等）に設置します。
3. アプリ内の設定で、テスト時の「グリーンスピード（Stimp値）」を固定しておきます（例：9.0ft）。

## テスト手順と記録方法
以下の3段階の強さでパットを打ち、それぞれ2〜3回ずつデータを記録してください。

* **短い距離**（例：約1mのパット）
* **中間の距離**（例：約2mのパット）
* **長い距離**（例：約3mのパット）

### 📝 記録する項目（各パットごと）
| 項目 | 取得元 / 確認方法 | 記録例 |
| :--- | :--- | :--- |
| **1. 実際のパット距離（実測値）** | マットの目盛りやメジャーで目視確認 | `2.0m` |
| **2. アプリの推定距離** | アプリ結果画面の大きな数字 | `1.8m` |
| **3. アプリの初速** | アプリ結果画面の小さな数字 | `1.62m/s` |
| **4. 記録されたコマ数** | フィルムストリップ上部に表示されるコマ数（※少なすぎないか確認用） | `12コマ` |
| **5. 設定Stimp値** | テスト時にアプリで設定した値 | `9.0 ft` |
| **6. スマホの設置状態** | どのような角度・高さで撮影したか（全体で1回記載すればOK） | `床に直置きして斜め下から撮影` |

## 分析・改善のアプローチ
取得したデータをもとに、以下のような分析と改善を行います。

1. **「実測距離」と「アプリの初速」が比例しているが、推定距離がズレる場合**
   👉 アプリの速度認識は正しいですが、物理モデル（摩擦係数）の計算式がマットの実際の抵抗と合っていません。Stimpモデルの計算式を環境に合わせてチューニングします。
2. **同じ強さ（実測距離）なのに、アプリの「初速」が毎回バラバラになる場合**
   👉 カメラによるボールのピクセル認識（遠近法パースや影の影響など）に誤差が生じています。基準となるスケール補正機能（キャリブレーション機能）や、回帰分析フィルタの強化などの改修を行います。
